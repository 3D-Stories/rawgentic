## Summary

The change adds deferred-to-target verification tracking across plan parsing, run-record validation, summary rendering, and workflow documentation. The main risk is that the new completion-gate helper does not enforce the same evidence fields the workflow claims are mandatory.

## Findings

### 1. [High] security · high confidence — hooks/plan_lib.py, assert_deferrals_recorded

> +        if not isinstance(e, dict) or not _is_nonempty_str(e.get("task_id")):
> +            errors.append(f"verification_deferred[{idx}] has no string task_id")
> +            continue
> +        recorded_ids.append(e["task_id"])

`assert_deferrals_recorded` only requires `task_id`, so a record entry like `{task_id: "2"}` can satisfy the completion-gate reconciliation even though the workflow says deferred verification requires `reason`, `local_proxy`, and `target_check`. That creates a fail-open path where a deferred task can be marked gate-satisfied without evidence of the required local proxy or target check.

**Recommendation:** Change `assert_deferrals_recorded` in `hooks/plan_lib.py` to validate every recorded entry has non-empty string `task_id`, `reason`, `local_proxy`, and `target_check`, matching `work_summary.validate_record`; add tests for missing `reason`, `local_proxy`, and `target_check` failing this helper.

### 2. [Medium] completeness · high confidence — skills/implement-feature/SKILL.md, completion-gate item 13

> +13. [ ] (conditional: any `plan_lib.deferred_tasks(tasks)` — verification deferred to target, #138) Every deferred task is recorded: `plan_lib.assert_deferrals_recorded(deferred_tasks(tasks), record["verification_deferred"])` returns `ok=True` (each entry carries reason + local_proxy + target_check), AND the PR body has the `## Deferred verification` section. Recorded ⇒ gate satisfied-with-note. An **unrecorded** deferral (planned deferred but missing/duplicate/foreign in the run-record list) ⇒ gate FAILURE — a deferral must never silently vanish into a pass.

The completion gate requires the PR body to have `## Deferred verification`, but the diff does not add any code-level check that the PR body actually contains that section. As written, the only implemented helper reconciles plan tasks against the run-record, so the documented PR-body requirement remains manual and can silently be missed.

**Recommendation:** Add a concrete completion-gate helper or existing gate integration that receives the PR body text and fails when `plan_lib.deferred_tasks(tasks)` is non-empty and the exact `## Deferred verification` heading is absent.

## Summary

The change adds a git worktree availability probe plus workflow documentation and tests. The main risk is that the new probe is documented as non-mutating and never-raising, but its cleanup path can still surface exceptions and can leave repository state behind when cleanup fails.

## Findings

### 1. [High] correctness · high confidence — hooks/capabilities_lib.py: finally cleanup in probe_parallelism

> +        if wt is not None:
> +            _git(["worktree", "remove", "--force", wt])
> +        if parent is not None:
> +            shutil.rmtree(parent, ignore_errors=True)
> +        _git(["worktree", "prune"])

The probe claims it never raises, but the finally block calls `_git(...)` outside any exception handling. If cleanup hits an OSError, missing `git`, inaccessible/deleted repo root, or another `subprocess.run` construction failure, that exception overrides the intended `worktree`/`serial-only` return. The concrete failure is that the new `probe-parallelism` command can crash instead of printing one of the two documented values.

**Recommendation:** In `probe_parallelism`, wrap the cleanup `_git(["worktree", "remove", ...])` and `_git(["worktree", "prune"])` calls in `try/except (FileNotFoundError, OSError, subprocess.SubprocessError)` or introduce a `_git_no_raise` helper for cleanup so cleanup failures cannot escape.

### 2. [Medium] correctness · high confidence — hooks/capabilities_lib.py: cleanup after worktree add

> +        if parent is not None:
> +            shutil.rmtree(parent, ignore_errors=True)
> +        _git(["worktree", "prune"])

The code ignores failure to delete the temporary worktree directory and then still prunes. Git `worktree prune` does not remove live worktree registrations whose paths still exist, so a failed `shutil.rmtree(..., ignore_errors=True)` can leave a registered throwaway worktree despite the stated cleanup guarantee. The concrete failure is stale worktree metadata and temp files left after a probe that otherwise reports success.

**Recommendation:** In `probe_parallelism`, after `shutil.rmtree`, verify `parent` no longer exists before pruning; if removal fails, run `git worktree remove --force wt` again or at least avoid claiming the repo is left unchanged. Add a test that simulates temp directory removal failure and asserts no registered worktree remains or the probe degrades cleanly.

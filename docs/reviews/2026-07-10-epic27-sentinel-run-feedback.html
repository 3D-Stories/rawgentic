What the gates actually bought — every run had a real catch

#24 · Step 11

A failed Prometheus subquery rendered reboot-required: none — a false all-clear — and an unreachable Alertmanager coerced to “up (0 peers)”. Both fixed pre-merge.

#26 · Step 8a, both reviewers

Unanchored REBOOT-REQUIRED grep became reachable through raw journal text: a log line merely containing the phrase could auto-reboot a guest at 3am. Reproduced, anchored.

#26 · Codex diff review

The cleanup lane’s journal vacuum ran before the error-count collection — silently shrinking the sweep window it was about to report on.

#23 · Step 11

$?-clobber and double-text bugs in the reboot-return handler — found after a fully green suite.

#20 · Step 11

Pinned exporter image tag didn’t exist on ghcr — the deploy would have been dead on arrival.

#25 · Codex diff review

Detached power-op runner could exit silently after the “shutting down…” ack on lock-open failure. Same spot a same-model reviewer scored 0.85 and the band dropped — cross-model disagreement earned its keep.

Recurring friction, ranked by frequency × cost → routing

  1. Step-16 record assembly is unvalidated manual work #361

    4 of 9 runs shipped wrong fields — goal_guard set-vs-deferred ×2, dispatches[] omitted despite well-formed lines ×2, complexity ×1, commit-count ×1, a gates row with no marker ×1 — and every one passed validate_record, which checks shape only.

  2. Marker emission tracks local reinforcement, not orchestrator quality #362

    Step 7’s marker missing in 7/9 runs; §7 has no local marker instruction while templated §11.5 was emitted every run. Markers are load-bearing for resume and for this very assessment.

  3. Usage telemetry effectively absent for epics #363

    7/9 runs capture_status “unavailable” (long multi-child sessions); one of the two captures is a hand-subtracted delta persisted as “captured”. No per-run attribution primitive exists.

  4. Mid-run mandate adoption has no reconcile step not filed

    The 3.25.0→3.27.1 cache bump landed mid-epic; DISPATCH compliance went 1-of-4 → 3-of-6 → 0 → 7/7 → 5/5. Enforcement half rides #361; adoption nudge noted for post_update_reconcile — revisit if it recurs.

  5. Resume vocabulary runs out at doc-only branches #364

    A design-checkpoint branch literally classifies to “changes” → resume at Step 9 with nothing implemented. #25’s correct resume was an inline judgment call, not the written words.

  6. Contaminated reviewer returns have no named handling #365

    #23’s reviewer cited a nonexistent SHA around a real finding; only orchestrator re-verification saved it. Dead-return detection exists; fabricated-citation returns don’t.

  7. Owner-gate / classifier pattern working as designed

    Every child hit auto-mode denials on host-privileged actions; all nine handled it correctly — no workaround, exact one-liners posted on the issue, honest deploy:manual.

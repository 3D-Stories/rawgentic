# #314 A/B experiment results — delegated-read index vs inline full-artifact read

Date: 2026-07-08. Owner acceptance bars: (1) B-arm saves tokens; (2) B-arm quality within
±5% pooled (no artifact >10% below). Design: `2026-07-08-314-delegated-read-digests.md`
§A/B experiment. Raw materials + sealed blinding key: session scratchpad `ab314/`.

## Verdict

| Bar | Result |
|---|---|
| **Quality ±5%** | **PASS** — pooled delta **+0.0000%** (A 9.0556 vs B 9.0556); worst artifact −3.39% (A3), well inside −10% |
| **Tokens saved** | **FAIL** — B consumed **+91.9%** MORE tokens than A (358,067 vs 186,601) |

Per the design's honest-failure clause the experiment result is reported as-is and the
ship decision returns to the owner. Not shipped as success.

## Setup (as designed)

3 real merged-PR diffs: A1 = 02c11bd (#316 telemetry, 19.8KB/7 files), A2 = f836259
(#311 wal-guard security fix, 29.8KB/11), A3 = 3488ab2 (#113 charter feature,
72KB/14). A-arm: one opus agent reads the full raw diff + answers a fixed 6-item
questionnaire. B-arm: sonnet index reader (fed the `--name-only` unit list) →
`plan_lib.validate_index` → opus consumer answers the SAME questionnaire from the index
+ ≤3 targeted `git show <sha> -- <file>` reads (never the full artifact). Scoring:
q1/q3 set-F1×10 vs git ground truth (identical arms — delta 0 on every artifact);
q2/q4/q5/q6 by per-artifact blinded opus judges, answers unlabeled in randomized
recorded order. Live adversarial probes against the validator: fabricated quote /
dropped fed file / vacuous return — **3/3 REJECTED**.

## Validator field results (independent of the bars)

- **Zero fabricated quotes** across 22 evidence entries from three real sonnet readers
  (every quote grep-verified against its artifact).
- **Zero coverage misses** — all three readers indexed exactly the fed unit set.
- **Sole rejection cause: the 120-char `one_line` cap** (A1: 5, A2: 1, A3: 18 entries
  over). Production behavior = inline fallback. A single prompt-level fix ("hard-wrap
  at 110 chars") or a cap of 160 would likely bring rejects to zero — worth one
  follow-up if the pattern ships.

## Why tokens failed (analysis, confirmed vs inferred)

- **Confirmed:** per-subagent fixed overhead (system prompt + tool schemas + reasoning)
  is ~30–50k tokens; B pays it twice (reader + consumer). At 20–72KB artifacts (~5–18k
  tokens of content), overhead dominates content by 3–10×, so adding an agent can never
  pay for itself at these sizes. Every reader ALONE cost more than its entire A-arm.
- **Confirmed (cost-weighted too):** even pricing sonnet at ~0.2× opus, B ≈ 61k
  opus-equivalent vs A ≈ 52k on A1 — still worse, because the opus consumer alone
  matches the whole A-arm.
- **Inferred (named limitation of this harness):** the A-arm is a one-shot subagent
  read; the PRODUCTION inline pattern the design targets is the ORCHESTRATOR holding a
  large artifact across ~8 Step-11 turns, re-charged as cache-read every turn
  (`usage_capture.py:104`) — the mechanism behind the #303 record's 86% orchestrator
  share. This harness could not measure held-context cost without running two full WF2
  runs; the deep-reasoner brainstorm predicted the win lives there, not in the one-shot
  read. What would confirm it: worker-share + orchestrator input tokens on the first
  two comparable post-reinstall WF2 runs, one with delegation active and one without.
- **Break-even estimate:** B beats A on raw tokens only when the artifact's token
  content exceeds the added agent overhead (~55k tokens ≈ 200KB+ artifact) — or when
  the inline alternative is a HELD read multiplied across turns. The shipped default
  thresholds (64KB diff / 32KB log) sit far below the one-shot break-even.

## Options for the owner (decision required)

1. **Ship prose+validator with thresholds raised to the empirical break-even**
   (e.g. `WF2_READ_DELEGATE_BYTES_DIFF=262144`): delegation then fires only on monster
   artifacts where it plausibly pays, quality-safe per this experiment. Cheap, honest.
2. **Ship as designed and measure the held-context claim in production** (AC4's
   deferred check on real WF2 runs): accepts interim token cost on mid-size diffs to
   test the real hypothesis this harness couldn't reach.
3. **Ship the validator + projections only** (projections are pure savings — no added
   agent), drop the two LLM reader surfaces until a held-context measurement exists.
4. **Abandon the reader surfaces entirely**, keep validator as tested infrastructure.

Recommendation: **option 1 + projections** — the quality result and validator field
results are genuinely positive; the failure is specifically "added agent overhead at
typical artifact sizes," which the threshold knob already controls.

## Round 2 — slim readers (owner-directed, 2026-07-09)

Re-run with reduced reader context (small-tool-surface agent proxy, single-pass paged-read
discipline, ≤110-char one_line brief) on the original three artifacts plus two
representative ones: **A4 = this branch's own diff** (80KB, over the 64KB trigger) and
**A5 = chorestory d958085** (128KB, cross-repo JS/SQL). Same blinded protocol, fresh
sealed key (`ab314/blinding-key2.json`).

| Artifact | A tokens | B r1 | B r2 (slim) | R2 gap | A quality | B2 quality |
|---|---|---|---|---|---|---|
| A1 (19.8KB) | 52,237 | 107,343 | 59,661 | +14.2% | 8.50 | 9.50 |
| A2 (29.8KB) | 57,717 | 112,325 | 67,983 | +17.8% | 8.17 | 8.17 |
| A4 (80KB) | 77,704 | — | 91,468 | +17.7% | 8.83 | 9.50 |
| A5 (128KB) | 83,291 | — | 142,260 | +70.8% | 9.33 | 9.33 |
| **Total/pooled** | **270,949** | (+91.9%) | **361,372** | **+33.4%** | **8.708** | **9.125** |

- **Quality: PASS with margin — B2 pooled +4.8% ABOVE control, worst artifact ±0.0%.**
  The index arm won A1 (F1 1.0 vs 0.5 on file ground truth) and A4 (surfaced a headline
  fact the full read missed), tied A2/A5.
- **Tokens: one-shot gap 91.9% → 33.4%** with only a PROXY slim reader (reviewer agent's
  4-tool surface; its reviewer-flavored system prompt still aboard — the purpose-built
  `rawgentic-reader` agent trims further). A5's +70.8% reflects real work: the reader
  disambiguated an embedded stale diff snapshot inside a review doc from the live hunks.
- **Validator, round 2: 2/5 clean; 3 rejects, ALL brief-grammar classes** (evidence
  quoted with the diff's +/− prefix; one_line cap overruns; locator annotations like
  "(new file)") — zero fabricated quotes in 8 reader runs across both rounds. Fix: the
  exact grammar lives in the dedicated reader agent's system prompt, not ad-hoc briefs.
- Reader baggage measured at 45.3k for generic agents (tool schemas dominate); consumers
  halved (49.6k → 24.7–39.1k) under the ≤3-targeted-reads discipline.

Round-2 conclusion: quality survives — improves — under the slim configuration; the
remaining one-shot premium is bounded and the carry-term economics (measured 99%
cache-read share, 165–254k/turn) sit on top. The reader-agent TODO is now also the fix
for every observed validator reject.

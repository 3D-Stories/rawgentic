# RCA — #334 codex thought-partner dispatches hang

## Hypotheses → evidence → root cause

**H1: The timeout-enforced consult code path is missing (issue's fix item 1).**
REFUTED. It already exists: WF13 `/rawgentic:peer-consult` (`skills/peer-consult/SKILL.md`)
over `hooks/adversarial_review_lib.py` — `run_codex_consult` (line 1161),
`build_consult_prompt` (1058), CLI `consult` subcommand (main(), exit contract
0/2/3/4, same 600s `RAWGENTIC_ADV_REVIEW_TIMEOUT`, fail-closed). Covered by
`tests/hooks/test_peer_consult_lib.py` (12 tests) + `test_peer_consult_registration.py`.
The issue as filed proposed building what is already shipped — the issue text needs a
scope correction comment.

**H2: Host sandbox breakage (AppArmor userns) made codex-rescue fail.**
CONFIRMED as the environmental trigger, ALREADY REMEDIATED at host level this session
(AppArmor profile `/etc/apparmor.d/bwrap` with `userns,`; verified
`codex sandbox bash -c 'echo ok'` → ok). Kernel-audit evidence in #334. Not an in-repo
defect — but the recipe lives only in an issue comment; nothing durable in the repo.

**H3: Routing/guidance gap — nothing directs cross-model consult work to the
sanctioned engine path.**
CONFIRMED — the actual in-repo root cause. Zero guidance anywhere in `docs/`,
`skills/`, or `shared/` mentions `codex-rescue` (only the audit report's count table).
An orchestrator (or a task brief) reaching for a cross-model thought partner has no
in-repo rule saying "use `/rawgentic:peer-consult` / the `consult` CLI — never a bare
`codex:codex-rescue` dispatch for load-bearing work," no dead-job protocol
(deadline; silent job = dead, mirroring #331's dead-agent rule), and no host runbook
for the userns failure signature. That is exactly how this session's two hangs
happened: the brief said "dispatch a codex:codex-rescue subagent," nothing contradicted
it, and the hang had no deadline.

## Root cause (selected)

H3, with H2 as the environmental trigger. A documentation/routing defect, not a code
defect: the reliable path exists and is tested; it is undiscoverable at the moment of
dispatch, and the failure mode of the unreliable path is undocumented.

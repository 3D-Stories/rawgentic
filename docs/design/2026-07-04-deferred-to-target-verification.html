Design: first-class deferred-to-target verification state (#138)

Date: 2026-07-04 · Issue #138 · Complexity: standard_feature (M — touches gate honesty) · lean gate

Codex passes folded — design (0 Critical / 2 High / 2 Medium) + Step-11 diff (0 Critical / 1 High / 1 Medium). All incorporated in place; no design loop-back (lean policy).

Problem

WF2's gates assume verification runs locally: TDD mode needs runnable tests, Implement-Verify needs a runnable command. Neither expresses "the dev env fundamentally cannot exercise this artifact" — an NSIS uninstaller with no makensis, native Win32 paste built from WSL for a Windows target, an OS-native tray menu that can't render headless (all from the SayStory campaign). Today that honest outcome ("compiles; must be verified on the target box") has no legible slot — it degrades into a hand-waved pass or an awkwardly skipped gate. Both dishonest.

Core invariant (anti-abuse)

Deferral must be impossible to use as a skip-everything lever. A deferred task STILL requires its best local proxy (compile, typecheck, unit tests of extractable logic); deferral covers only the unexercisable remainder, which never counts as verified. The gate is satisfied only when the deferral is recorded with evidence (reason + what-was-run-locally + the manual target check); an unrecorded or evidence-less deferral is a gate FAILURE. A proxy you can run is never the path you can't — so the gap is named, never papered over.

Approach

1. Plan contract (plan_lib)

  • Task.deferral_reason parsed from - verification: deferred-to-target (<reason>). Additive; absent → None (fail-open); a deferral line with no (reason) → PlanFormatError (fail-closed). Free-text - verification: lines stay ignored (no existing parser consumes them).
  • deferred_tasks(tasks) → the deferred subset.
  • assert_deferrals_recorded(plan_deferred, recorded_entries) → the mechanical completion-gate check: every plan-deferred id appears exactly once, and every entry carries non-empty task_id + reason + local_proxy + target_check [Step-11 F1 — an evidence-less entry must not satisfy the gate]. Missing/duplicate/foreign/evidence-less → fail.
  • assert_pr_body_has_deferred_section(pr_body, plan_deferred) → fails when deferrals exist but the canonical ## Deferred verification heading is absent [Step-11 F2].

2–5. Across the spine (SKILL.md + work_summary + run-record.md)

  • Step 5: declare deferral when the env can't exercise the artifact (best local proxy still required).
  • Step 9 Part B: list each deferred task + the local proxy that WAS run; never counted as verified; a deferred task with no proxy evidence is not satisfied.
  • Step 12: a ## Deferred verification PR section (one canonical heading everywhere [F4]), omitted when empty, generated from the same list.
  • Step 16 run-record: verification_deferred is a structured list [{task_id, reason, local_proxy, target_check}] [design-F1 — a bare count can't tell which tasks were recorded or whether evidence exists]. Validated only when present; old records stay valid. Rendered per item.
  • <completion-gate>: calls both helpers; recorded-with-evidence + PR section ⇒ satisfied-with-note; any gap ⇒ FAILURE.

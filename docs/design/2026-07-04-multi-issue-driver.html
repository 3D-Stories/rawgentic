Design: autonomous multi-issue driver (#134)

Date: 2026-07-04 · Issue #134 · design-only — this issue is the design gate, not the build.

Codex adversarial pass folded — 0 Critical / 1 High / 3 Medium, all incorporated in place (no design loop-back per the lean review policy).

Problem

WF2 is per-issue by contract: <termination-rule> ends at Step 16 and forbids auto-continuation. An autonomous backlog (N issues, push/PR/merge each) means re-invoking the 16-step skill N times with no shared state. This very campaign (six issues merged) hand-rolled every missing piece:

Hand-rolled this campaignWhere it livedFailure risk when improvised
Ordered queue with per-issue statusimpact-order list in the handoffdrift between "what's next" and reality
Rollback anchor per issuebase SHA at branch creationnone captured → can't cleanly revert a bad merge
Alignment ledgercampaign-131-140.handoff.mdlost across compaction if not written
Inter-issue policy (review budget, never-Haiku)operator memory + standing rulessilently forgotten mid-run
DEFER (park + keep going)never existedan owner-reserved surface would stall the whole loop

Two real bugs trace to the missing structure: #138 was committed to local main (no driver enforced "branch before first commit"); #140 fixed a Step-7 base bug that only bites in a multi-issue loop (branching off a stale sibling). Both are exactly what a driver prevents.

Decision (AC2): documented PATTERN + a small committed state file — NOT a new skill

Recommendation: documented pattern. Why the skill alternative loses:

  • A skill adds ceremony where the agent is already competent. The looping — run WF2, advance on success, park on DEFER, repeat — is control flow an orchestrator runs reliably without a skill (six issues, zero loop-control errors this campaign).
  • The value is in CONVENTIONS (data + docs), not executable logic: the queue schema, DEFER taxonomy, rollback-anchor protocol, resumption contract. Those are a pattern doc + one JSON state file, optionally a thin pure validator — not a 16-step skill.
  • A skill would be tempted to re-implement or weaken WF2's gates. A pattern that invokes /rawgentic:implement-feature fresh per issue makes weakening WF2 structurally impossible — AC3.

Ships (in the follow-up build, not here): docs/multi-issue-driver.md; a committed queue state file claude_docs/.driver-state/<campaign>.json; OPTIONAL thin hooks/driver_lib.py pure validators, added only if hand-maintained JSON proves error-prone.

Queue schema (AC1)

Per-campaign claude_docs/.driver-state/<campaign>.json: policy {order, deploy, has_deploy, smoke_gate, review_budget, never_haiku}, base_default_branch_sha, and an issues array with {number, status, pr, merge_sha, rollback_anchor, deferred_reason, deferred_branch, branch_preservation}.

Status transitions (folded F1 — pr_open is first-class so headless is representable):

queued → in_progress → { pr_open | merged | deferred | abandoned }
pr_open → { merged | deferred | abandoned }   # headless stops at pr_open

One issue in_progress/pr_open at a time (serial; parallel needs worktrees, #136/#85).

DEFER taxonomy (AC1)

Branch preservation on DEFER (folded F2 — deterministic + persisted): branch_preservation ∈ {pushed, discarded, none} with deferred_branch. Resumable deferrals with commits → pushed; abandoned approach → discarded (branch deleted); no commits → none. Resumption never assumes a discarded branch or loses pushed work.

Rollback-anchor protocol (AC1)

Capture origin/<default> SHA as each issue's rollback_anchor before its branch is created (this campaign captured it implicitly; the driver persists it). Bad merge → git revert <merge_sha>. The anchor also asserts the next issue branches from a fresh base (the #140 fix), not a stale sibling.

Deploy policy (AC1)

per-issue (deploy + smoke each merge; safest, slowest), batch (deploy once at end), none (libraries / PR-terminal — this campaign). has_deploy + smoke_gate copied from capabilities into the state file (folded F4); deploy: per-issue with has_deploy: false is a rejected config error.

Resumption (AC1) — reconciliation table (folded F3)

The committed state file is the resumption substrate. Observed remote state wins over the stale queue value:

Real gh/git stateReconciled action
PR mergedmark merged, advance
PR open, CI greennon-headless → merge + advance; headless → leave pr_open, stop
PR open, CI red/pendingresume WF2 at Step 13
no PR, branch has commitsresume WF2 at Step 9/11 per resume_lib
no PR, empty branchresume WF2 at Step 8
no branchrestart at WF2 Step 7 (fresh base)
branch_preservation: discardedhonor the deferral, do not seek the branch
branch_preservation: pushedre-open per deferral reason when unblocked

The driver decides only the driver-layer action; intra-WF2 resume is delegated to WF2's own resume_lib so the driver never re-implements step detection.

<termination-rule> interplay (AC3 — not weakened)

The driver never re-enters or extends a WF2 run. Each iteration invokes /rawgentic:implement-feature fresh for one issue; that run terminates at Step 16 with all its gates. The driver observes the outcome and advances its own queue — it owns queue/deferral/anchor/policy, never a WF2 step.
